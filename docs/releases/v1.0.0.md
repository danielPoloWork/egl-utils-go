# v1.0.0 — Feature-complete: the full utilities library

**Date:** 2026-07-15 · **Milestones:** M2–M9 (M1 shipped in v0.1.0)

The first stable release. Every one of the 25 specified features is implemented under the full
quality bar, and `v1.0.0` commits to Semantic Versioning API stability for the whole exported
surface: no breaking change to these signatures or their documented behaviour without a MAJOR bump.

## What's in it

- **Concurrency (M2)** — `workerpool`, `pubsub`, `fanin`, `fanout`, `semaphore`: channel-native
  primitives, leak-free (goleak-gated) and race-clean.
- **Resilience (M3)** — `circuitbreaker`, `retry`, `ratelimit`: timerless/goroutine-free state
  machines with deterministic-clock tests and zero-allocation hot paths.
- **HTTP middleware (M4)** — `middleware.RequestID`, `Logger`, `Recoverer`, `Cors`: a composable
  decorator chain across the public HTTP edge, with the trust boundary threat-modelled (controls
  C-2, C-3).
- **Configuration (M5)** — `config.Load[T]` (JSON/YAML + `${VAR}` expansion + `Validator`),
  `env.GetDefault` and typed variants.
- **Structured logging (M6)** — `logger.NewStructured` (JSON slog for ES/Loki) and
  `logger.WithFields`/`FromContext` (context-carried fields).
- **Caching & data (M7)** — `cache.InMemory` (generic TTL cache, Get-enforced expiry, one bounded
  sweeper) and `db.Transaction` (commit/rollback/panic-safe).
- **Validation & security (M8)** — `validator.Struct` (tag-driven, hand-rolled) and
  `hash.HashPassword`/`CheckPassword` (bcrypt; control C-4, security-auditor sign-off).
- **Diagnostics & lifecycle (M9)** — `lifecycle` (ordered graceful shutdown), `health.Handler`,
  `metrics.Prometheus`, `syncpool.BufferPool` (zero-alloc), `errors.Wrap`/`Wrapf`
  (stack-preserving).

## Quality & governance

- Runtime dependencies held to ADR-0004's budget: stdlib + `golang.org/x/*`
  (`sync`, `crypto`) + two vetted third parties (`gopkg.in/yaml.v3`, `prometheus/client_golang`).
  The Go 1.24 floor is preserved (every dependency pinned to a floor-compatible release);
  `govulncheck` reports no called vulnerabilities.
- Every feature carries an ADR (0005–0029); security-relevant decisions additionally carry
  threat-model rows and compliance controls (C-1…C-4).
- Full gate green: build matrix (Go 1.25 & 1.26, Linux/Windows/macOS), gofumpt, golangci-lint v2
  (govet, staticcheck, errcheck, revive, gosec), `go test -race`, govulncheck, benchmarks, and the
  cross-artifact consistency lint.

## Compatibility

Module `github.com/danielPoloWork/egl-utils-go`, one import path per feature package. From v1.0.0,
the exported API is stable under SemVer.
