# 2026-07-15 — Milestone 4.4: HTTP middleware (Cors) — Milestone 4 complete

## What got done

- **Roadmap 4.4 `middleware.Cors`** (branch `feat/middleware-cors`, draft PR #20, ADR-0017, patterns row 9):
  the fourth and final HTTP middleware — **Milestone 4 is complete** (RequestID, Logger,
  Recoverer, Cors). Spec §5 froze the constructor `Cors(cfg CorsConfig) func(http.Handler)
  http.Handler`; the `CorsConfig` shape was the design work (ADR-0017).
- **`CorsConfig` — six fields, deny-by-default.** `AllowedOrigins/Methods/Headers/ExposedHeaders
  []string`, `AllowCredentials bool`, `MaxAge time.Duration`. The zero value allows no origin
  (safe default). Three request paths: no `Origin` → forwarded untouched; **preflight** (OPTIONS +
  `Access-Control-Request-Method`) → answered directly with **204** and the negotiated headers,
  never forwarded; actual request → forwarded, with `Access-Control-Allow-Origin` added when the
  origin is allowed and omitted when not (the browser enforces, not us).
- **Security teeth (why the ADR):** two misconfigurations **panic at construction** —
  `AllowCredentials` with a `*` origin (Fetch-spec violation; reflecting every origin with
  credentials is a whole-web open door) and a negative `MaxAge`. A specific allowed origin is
  **echoed with `Vary: Origin`**; `*` is emitted only without credentials. `AllowedHeaders`
  empty/`*` reflects the request's `Access-Control-Request-Headers`; `AllowedMethods` empty defaults
  to `GET, HEAD, POST`. Subdomain/regex origin patterns deliberately deferred (additive later).
  New **compliance control C-3** (CORS origin & credential policy) + a threat-model Cors row.
- **Implementation note:** `Cors` precomputes an unexported `cors` struct (joined header strings,
  origin set) once at construction; the per-request path only writes headers. `writeAllowOrigin`
  returns whether the response varies on Origin so each path adds `Vary: Origin` exactly once (no
  duplicates between the actual and preflight paths).

  coverage**, gofumpt clean, golangci-lint v2 (`/v2` path) 0 issues, govulncheck clean,
