# 2026-07-15 — Milestone 9.3: metrics.Prometheus

## What got done

- **Roadmap 9.3 `metrics.Prometheus` / `metrics.Handler`** (branch `feat/metrics-prometheus`,
  ADR-0027): Prometheus HTTP instrumentation — the third of Milestone 9 (3 of 5), and the milestone
  that brings in **the last of ADR-0004's two budgeted third-party runtime deps**. Spec §5 froze
  `Prometheus(reg prometheus.Registerer) func(http.Handler) http.Handler` and `Handler() http.Handler`.
- **Metrics & bounded cardinality (the review point):** a `http_requests_total` counter and a
  `http_request_duration_seconds` histogram (DefBuckets), each labelled **(method, code) only**.
  **The path is never a label** (unbounded → the classic Prometheus memory bomb, and attacker-driven
  via arbitrary URLs). **The method label is normalized** to the nine known HTTP methods + `"other"`,
  because `r.Method` is client-controlled and arbitrary tokens would otherwise be an unbounded,
  attacker-reachable cardinality vector. Recorded as a threat-model DoS row.
- **API shape:** `Prometheus(reg)` registers the collectors on the caller's registry via
  `MustRegister` (double-install panics — loud wiring error); `Handler()` returns `promhttp.Handler()`
  over the default registry. Documented pairing: `Prometheus(prometheus.DefaultRegisterer)` + `Handler()`;
  custom registry → `promhttp.HandlerFor` directly. Status captured by an `Unwrap`-aware
  `statusRecorder` (same technique as middleware, re-implemented since the type is unexported).
  Loud panic on nil registerer / nil handler (ADR-0005).
- **Dependency: `github.com/prometheus/client_golang` v1.23.2** (ring 3, ADR-0004 pre-approved). Its
  `go` directive is **1.23.0** — under the 1.24 floor, so **no floor gymnastics** (unlike x/crypto).
- **Dependency-hygiene decision worth remembering:** adding prometheus pulled transitive
  `golang.org/x/sys@v0.41.0`, which carries **GO-2026-5024** (integer overflow in
  `x/sys/windows.NewNTUnicodeString`). `govulncheck` confirms it is **uncalled** (0 called vulns).
  Its fix is only in `x/sys@v0.44.0`, whose `go` directive is **1.25.0** → bumping would **drop the
  1.24 floor**. **Decision: keep v0.41.0, preserve the floor; accept the uncalled advisory, revisit
  at the 1.25 floor move.** The floor is worth more than clearing an unreachable finding. (Lesson:
  "clear every advisory" is not free — weigh it against the floor.)
- Local gauntlet green (portable Go 1.26.5): `go mod tidy`, build, vet, full `go test ./...`,
  **100% metrics coverage**, gofumpt clean, golangci-lint v2 0 issues, govulncheck (0 called),
  `consistency_lint.py` OK. `-race` is CI-only locally (no cgo compiler).
