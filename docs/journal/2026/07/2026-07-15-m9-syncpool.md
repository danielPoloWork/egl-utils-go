# 2026-07-15 — Milestone 9.4: syncpool.BufferPool

## What got done

- **Roadmap 9.4 `syncpool.BufferPool`** (branch `feat/syncpool-bufferpool`, ADR-0028): a pooled
  `*bytes.Buffer` reuse type — the fourth of Milestone 9 (4 of 5), and the concrete realization of
  spec §1's `sync.Pool` object-reuse pillar. Spec §5 froze `NewBufferPool() *BufferPool`,
  `Get() *bytes.Buffer`, `Put(*bytes.Buffer)`.
- **The two ways a naive sync.Pool wrapper goes wrong, both handled:**
  - **Dirty buffers:** `Put` calls `Reset()` before pooling, so `Get` always hands out an empty
    buffer (`Len()==0`) — a pool returning dirty buffers would leak one caller's bytes to the next.
  - **Unbounded retention (the real trap):** `Put` **discards** a buffer whose capacity has grown
    past a 64 KiB cap instead of pooling it. `sync.Pool` keeps whatever you return, so one request
    that grows a buffer to megabytes would pin that capacity forever and steady-state memory would
    ratchet up under bursty large payloads. The cap bounds steady-state memory; the occasional large
    buffer is served, used, and dropped to the GC.
- **Zero-allocation, proved two ways:** a `testing.AllocsPerRun` assertion of **0 allocs** for a
  Get/write/Put cycle, plus a co-located benchmark (~17 ns/op serial, ~8 ns parallel, 0 allocs/op;
  house style — `ReportAllocs`, `b.N`/`RunParallel`). A nil `Put` is a no-op (proportionate — no
  downstream hazard, unlike a nil dependency).
- **Object Pool pattern** flips **Planned → Implemented** in the catalogue (row 10) with ADR-0028 and
  `syncpool/` as its code location.
- Tests (100% coverage): empty-on-Get, reset-and-reuse (pointer identity, single goroutine), nil Put,
  the AllocsPerRun proof, an 8×500 concurrent hammer (the `-race` CI target), and — in an **internal**
  test so it can reference `maxRetainedCap` precisely — the oversized-discard and within-cap-retain
  behaviors.
- Local gauntlet green (portable Go 1.26.5): build, vet, full `go test ./...`, 100% syncpool
  coverage, gofumpt clean, golangci-lint v2 0 issues, govulncheck (0 called), `consistency_lint.py`
  OK (Object Pool row now cites an existing ADR + code loc). `-race` is CI-only locally.
