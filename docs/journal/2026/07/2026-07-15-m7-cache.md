# 2026-07-15 — Milestone 7 opens: cache.InMemory

## What got done

- **Roadmap 7.1 `cache.InMemory`** (branch `feat/cache-inmemory`, ADR-0021): the first of
  Milestone 7 and the first component since M2 to **own a goroutine** — back to the
  concurrency-critical tier (Fable 5 · high per the ROADMAP tag). Spec §5 froze
  `NewInMemory[K comparable, V any](ttl time.Duration, opts ...Option) *Cache[K, V]`,
  `Get`/`Set`/`Delete`, `Close`, `ErrNotFound`.
- **The design's center of gravity: expiry is enforced by `Get`, not the sweeper.** Every entry
  stores its deadline; `Get` compares at call time (deadline exclusive — expired *at* the deadline)
  and returns `ErrNotFound` the instant it passes. The periodic sweeper is **purely a memory
  reclaimer** (`WithCleanupInterval`, default ttl) — the classic TTL-cache stale-read window between
  deadline and sweep is impossible by construction.
- **Goroutine lifecycle, M2 discipline:** exactly one sweeper (ticker + `done` channel), stopped
  deterministically by a `sync.Once` `Close` (idempotent, concurrent-safe, immediate mid-interval;
  ticker stopped on exit). goleak gates every test, including a dedicated
  `TestCloseStopsSweeper` where `VerifyNone` runs immediately after a mid-interval `Close`.
  **After `Close` the cache stays usable** (Get still refuses expired entries) — graceful like
  `pubsub.Close` (ADR-0006), not loud like `workerpool.Stop`, because a closed cache has no
  observable failure mode to make loud. Loud panics on non-positive ttl/interval (ADR-0005).
- **Deterministic expiry tests** via the injectable unexported `now` (ADR-0010 idiom): the boundary
  is asserted exactly (live at deadline−1ns, expired at deadline) with no sleeps; background
  reclamation tested separately with a real clock and tight interval.
- **Zero-allocation hot paths** (spec §1): Get-hit ~28 ns, Get-miss ~15 ns, Set ~51 ns, all
  **0 allocs/op** (co-located benchmarks, house style — `ReportAllocs`, `b.N`/`RunParallel`).
- Test-ordering lesson (caught locally): a `t.Cleanup(c.Close)` registered *after*
  `defer goleak.VerifyNone(t)` runs **after** goleak, which then sees the live sweeper. Fixed by
  deferring `Close` inside the test after the goleak defer (defers are LIFO). Worth remembering for
  every goroutine-owning component.
- Local gauntlet green (portable Go 1.26.5): build, vet, full `go test ./...`, **100% cache
  coverage**, gofumpt clean, golangci-lint v2 0 issues, govulncheck clean, `consistency_lint.py` OK.
  `-race` is CI-only locally (no cgo compiler) — the Linux race job is the gate for the sweeper.
