# 2026-07-15 — Milestone 5 opens: config.Loader

## What got done

- **Roadmap 5.1 `config.Load[T]`** (branch `feat/config-loader`, ADR-0018): the first of
  Milestone 5 and the module's first configuration surface. Spec §5 froze
  `Load[T any](path string, opts ...Option) (T, error)`. Decodes a JSON or YAML file (format
  chosen by extension: `.json` / `.yaml` / `.yml`; anything else → sentinel
  `ErrUnsupportedFormat`) straight into the consumer's `T` — no intermediate map.
- **Environment via `${VAR}`/`$VAR` expansion** (`os.Expand` + `os.Getenv`), on by default,
  applied to the file bytes before parsing so secrets/per-env values stay out of the committed
  file; `WithoutEnvExpansion()` disables it. Field-level env overlay by struct tag is deliberately
  deferred (additive); direct typed env reads are 5.2 `env.GetDefault`'s job.
- **Validation via a `Validator` interface** (`Validate() error`) — called through `*T` so a
  pointer-receiver method is found; the richer tag-driven `validator.Struct` (M8) composes on top
  later. Functional options (ADR-0005 idiom) keep the frozen signature stable.
- **YAML parser selected + pinned: `gopkg.in/yaml.v3`** (ADR-0018), closing the review point
  ADR-0004 deferred to this PR. It was already an indirect dependency (via testify), so this only
  promotes it to direct — **no new supply-chain surface**; C-1's dependency ring is unchanged.
- Local gauntlet green (portable Go 1.26.5): `go mod tidy`, build, vet, full `go test ./...`,
  **100% config coverage**, gofumpt clean, golangci-lint v2 (`/v2` path) 0 issues, govulncheck
  clean, `consistency_lint.py` OK. `-race` is CI-only locally (no cgo compiler).
