## How the next session resumes

Per `docs/workflow/release.md` §8, once the maintainer **merges** the release PR, the **agent**
creates and pushes the annotated tag — `git tag -a v1.0.0 -m "<headline>"` then
`git push origin v1.0.0` — which lets CI open the GitHub Release as a **draft**; the maintainer then
clicks **Publish** (the human checkpoint). The agent never publishes, never amends a published tag.
So the next action after merge is: **tag v1.0.0 and push the tag** (do not tag before the PR is
merged to master). Portable Go under `%TEMP%\go-portable`; `/v2` golangci-lint path; `-race` CI-only.
