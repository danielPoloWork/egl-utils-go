# 2026-07-15 — Milestone 7.2: db.Transaction — Milestone 7 complete

## What got done

- **Roadmap 7.2 `db.Transaction`** (branch `feat/db-transaction`, ADR-0022): the auto-rollback
  transaction helper, the second and final item of Milestone 7 — **M7 (Caching & data helpers) is
  complete**. Spec §5 froze `Transaction(ctx, db *sql.DB, fn func(*sql.Tx) error) error`.
- **Three finalize paths, one finalization each** (the review point was the panic path):
  - `fn` returns nil → **commit** (commit error wrapped).
  - `fn` returns err → **rollback, return err**; if the rollback itself fails it is joined via
    `errors.Join` (neither cause nor cleanup failure lost), except `sql.ErrTxDone` which is treated
    as a no-op (a self-finalizing callback produces no spurious error).
  - `fn` panics → **rollback in a deferred recover, then re-panic the original value** unchanged, so
    the caller's `recover`/`Recoverer` sees exactly what `fn` threw. Rollback is best-effort; a
    failed rollback never masks the panic (mirrors Recoverer, ADR-0016, and the stdlib example).
- **Context-governed:** `BeginTx(ctx, nil)` ties the tx to `ctx`; an already-cancelled context fails
  the begin (`errors.Is(err, context.Canceled)`) before `fn` runs. Loud panic on nil db / nil fn
  (ADR-0005). Composes with `retry.Backoff` (retryable tx) and `Recoverer` (handler panic → rollback
  → clean 500).
- **Testing without a real database:** a minimal in-repo `database/sql/driver` fake wired through
  `sql.OpenDB` + a `driver.Connector` — no global `sql.Register` (no cross-test name collisions),
  and **no `sqlmock`** (ADR-0004 permits only testify/goleak/rapid as test deps). Injectable
  begin/commit/rollback/exec errors + call counters cover every path: commit, error-rollback,
  panic-rollback-repanic, begin error, commit error, joined rollback error, a real statement via
  `tx.ExecContext`, cancelled context, and both nil-wiring panics.
- Local gauntlet green (portable Go 1.26.5): build, vet, full `go test ./...`, **100% db coverage**,
  gofumpt clean, golangci-lint v2 0 issues, govulncheck clean, `consistency_lint.py` OK. `-race` is
  CI-only locally (no cgo compiler).
- Lint lesson: errcheck flags a bare `defer sdb.Close()` — used a `closeDB(t, sdb)` helper that
  asserts the error (also deferred after the goleak defer so it runs first, LIFO — same sql
  background-goroutine ordering care as the cache sweeper).
