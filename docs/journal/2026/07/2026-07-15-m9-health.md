# 2026-07-15 — Milestone 9.2: health.Handler

## What got done

- **Roadmap 9.2 `health.Handler`** (branch `feat/health-handler`, ADR-0026): a preconfigured
  dependency-probing HTTP health endpoint — the second of Milestone 9 (2 of 5). Spec §5 froze
  `Handler(checks ...Check) http.Handler` with `Check{Name string, Probe func(ctx) error}`.
- **Concurrent probes on the request context.** Each request runs every probe in parallel with
  `r.Context()`, so endpoint latency is the slowest probe (not the sum) and cancellation/deadline
  propagate; `run` waits for all (a `WaitGroup`) so **no goroutine outlives the response** (spec §1).
  200 when all pass, 503 when any fails; no checks = healthy (200, bare liveness).
- **Security decision — status-only body, error never leaked.** The JSON is
  `{"status":"ok"|"unavailable","checks":{name:"ok"|"fail"}}`; the probe's **error text is
  deliberately omitted**, because `/healthz` is commonly unauthenticated and a probe error carries
  internal detail (DSNs, hostnames). The failing check's name locates the problem; a consumer wanting
  the error logs it inside the probe. Same no-detail-to-client posture as Logger (ADR-0014) and
  Recoverer (ADR-0016) — recorded as new threat-model Information-disclosure and DoS rows on the HTTP
  edge. A dedicated test (`TestErrorTextIsNotLeaked`) asserts a fake DSN in a probe error never
  reaches the body.
- Loud panics on empty/duplicate check name or nil probe (ADR-0005); the checks slice is copied so
  later caller mutation can't affect the handler. Deterministic output (encoding/json sorts map keys).
- Tests (100% coverage): all-healthy, one-unhealthy, no-checks, error-not-leaked, request-context
  propagation, a **concurrency proof** (N probes each block on a shared latch until all have started —
  guarded by a 2s timeout so a serial regression fails rather than hangs), and the panic table. The
  `-race` CI job covers the results-map mutex.
- Lint note: gosec **G101** flagged the fake DSN test fixture as "hardcoded credentials" — suppressed
  with a single-line `//nolint:gosec` + reason (per `.golangci.yml`; the fixture exists precisely to
  prove the error is *not* leaked).
- Local gauntlet green (portable Go 1.26.5): build, vet, full `go test ./...`, 100% health coverage,
  gofumpt clean, golangci-lint v2 0 issues, govulncheck (no called vulns), `consistency_lint.py` OK.
  `-race` is CI-only locally (no cgo compiler).
