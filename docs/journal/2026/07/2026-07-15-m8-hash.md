# 2026-07-15 — Milestone 8.2: hash (bcrypt) — Milestone 8 complete

## What got done

- **Roadmap 8.2 `hash.HashPassword` / `hash.CheckPassword`** (branch `feat/hash-password`,
  draft PR #28, ADR-0024): bcrypt password hashing and verification — the **security-relevant** half of Milestone
  8, which **M8 (Validation & security) now completes**. Spec §5 froze the two signatures; §2.20
  names bcrypt. Carries ADR-0024, compliance control **C-4**, a threat-model **credential store**
  boundary + two STRIDE rows, and needs the **security-auditor sign-off** (AGENTS.md §10).
- **Design (all security decisions in ADR-0024):**
  - **bcrypt at default cost 10** — the frozen signature has no cost knob; 10 is bcrypt's default and
    within OWASP range; a higher cost trades login latency / DoS surface. Configurable cost deferred.
  - **Per-hash random salt** embedded in the output (no salt column); each call yields a distinct
    hash, both verify (tested).
  - **72-byte limit → `ErrPasswordTooLong`**, never silent truncation (which would collide distinct
    long passwords).
  - **Constant-time verify** (`bcrypt.CompareHashAndPassword`); wrong password → generic
    `ErrMismatch`; malformed hash → distinct wrapped error. godoc directs callers to surface only a
    generic failure (no user enumeration).
  - **Self-contained errors:** `ErrMismatch` is our own (translated); `ErrPasswordTooLong` is
    bcrypt's, re-exported — callers `errors.Is` against `hash.*` without importing bcrypt.
- **Dependency: `golang.org/x/crypto` v0.48.0** — the module's first new runtime dep since yaml.v3,
  ring 2 (pre-approved for bcrypt by ADR-0004). **Pinned to v0.48.0 deliberately: it is the newest
  release whose `go` directive is still `1.24.x`; v0.50.0+ declare `go 1.25.0` and would raise the
  module floor.** Adding it normalized our directive `go 1.24` → `go 1.24.0` (same 1.24 floor, made
  explicit) — the exact floor-preserving discipline ADR-0009 used for x/sync. `go.sum` committed;
  `govulncheck ./...` reports **0 called vulnerabilities** (only `bcrypt` is imported; advisories in
  x/crypto's unused packages such as `ssh` are not reachable).
- Lint note: revive flags `hash.HashPassword` as a stutter, but the name is frozen by spec §5, so a
  single-line `//nolint:revive` with an in-line reason (per `.golangci.yml`). Coverage reached 100%
  by re-exporting bcrypt's `ErrPasswordTooLong` (collapsing HashPassword to one error branch) rather
  than an unreachable defensive `else`.
- Local gauntlet green (portable Go 1.26.5): build, vet, full `go test ./...`, **100% hash
  coverage**, gofumpt clean, golangci-lint v2 0 issues, govulncheck (no called vulns),
