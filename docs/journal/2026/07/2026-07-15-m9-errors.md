# 2026-07-15 — Milestone 9.5: errors.Wrap — feature-complete

## What got done

- **Roadmap 9.5 `errors.Wrap` / `errors.Wrapf`** (branch `feat/errors-wrap`, draft PR #33, ADR-0029): stack-
  preserving error-context helpers — **the final roadmap item. The library is now feature-complete:
  all 25 spec features are implemented.** Spec §5 froze `Wrap(err error, msg string) error` and
  `Wrapf(err, format, args...) error`, requiring `errors.Is`/`As`/`Unwrap` transparency.
- **`%w`-transparent:** `*wrapped.Unwrap()` returns the cause, so `errors.Is`/`As`/`Unwrap` see
  through to every error below; `Error()` renders `"message: cause"`. Verified against a sentinel
  (Is) and a typed error (As).
- **One captured stack per chain, at the origin.** The first wrap of a stackless chain runs
  `runtime.Callers`; **every later wrap reuses that same stack by reference** (found via the inner
  `StackTracer`), so `runtime.Callers` runs exactly once and the trace always points at the original
  failure site — "without losing the original call stack" (spec §2.25). Each `wrapped` stores the
  origin stack, so `StackTrace()` is a trivial accessor (this also removed a defensive-unreachable
  read-path branch → 100% coverage without dead code).
- **Stack exposed, not hidden:** exported `StackTracer` interface (`StackTrace() []uintptr`, composes
  with `runtime.CallersFrames`) and `fmt.Formatter` — `%+v` prints the frames, `%v`/`%s` the message,
  `%q` the quoted message.
- **`Wrap(nil, …)` / `Wrapf(nil, …)` return nil** — a helper must not manufacture an error from the
  absence of one (so an unconditional `return errors.Wrap(err, …)` is correct).
- **Package name `errors` shadows the stdlib:** the spec froze the path `.../egl-utils-go/errors`;
  internally it imports the standard library as `stderrors` for `errors.As`, and the godoc tells a
  consumer needing both to alias one.
- Local gauntlet green (portable Go 1.26.5): build, vet, full `go test ./...`, 100% errors coverage,
  gofumpt clean, golangci-lint v2 0 issues, govulncheck (0 called), `consistency_lint.py` OK. `-race`
  is CI-only locally.
